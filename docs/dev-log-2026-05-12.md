## 아키텍처

```
[Browser]
    │
    ▼
[Nginx Gateway :8080]
    ├── /api/auth/*       → Auth Service  (FastAPI  :8000)
    ├── /devticket/api/*  → Backend       (Spring   :8082)  ← auth_request 검증
    └── /devticket/*      → Frontend SPA  (Nginx    :80)

[PostgreSQL :5432]  ← Auth Service + Backend 공용 DB
```

### 인증 흐름

```
Request → Nginx
    └── auth_request /internal/auth/verify  → Auth Service (쿠키 검증)
            │  200 OK + X-User-* 헤더 반환
            ▼
        Backend (AuthInterceptor)
            └── X-ERP-User-Id / X-ERP-User-Role / X-ERP-Display-Name 등을
                CurrentUser(ThreadLocal)에 바인딩 → 컨트롤러에서 사용
```

## 티켓 상태 흐름

```
대기(1) ──[픽업]──▶ 진행중(2) ──[QA요청]──▶ QA검증(3)
                        │                        │
                  [보류*·반려*]           [재작업*] [완료]
                        ▼                        ▼
                   보류(5)·반려(6)            완료(4)
                                                 │
                                        [재오픈*, 7일 이내]
                                                 ▼
                                             대기(1)

 * 사유 입력 필요
```

| 전환 | 실행 주체 | 사유 |
|------|-----------|------|
| 대기 → 진행중 (픽업) | DEVELOPER, ADMIN | 불필요 |
| 진행중 → QA검증 | DEVELOPER, ADMIN | 불필요 |
| 진행중 → 완료 | DEVELOPER, ADMIN | 불필요 |
| 진행중 → 보류 | DEVELOPER, ADMIN | **필요** |
| 진행중 → 반려 | DEVELOPER, ADMIN | **필요** |
| QA검증 → 재작업 (진행중) | REQUESTER | **필요** |
| QA검증 → 완료 | REQUESTER | 불필요 |
| 완료 → 대기 (재오픈) | 모든 역할 | **필요** |

## DB 스키마 요약

```
users ─── groups (FK: group_id)
  │     └── positions (FK: position_id)
  │     └── roles (FK: role_id)
  │
  └── dts_tickets (requester_id, assignee_id → users.id)
          │
          ├── dts_ticket_history  (ticket_id, changed_by_id)
          ├── dts_ticket_watchers (ticket_id, user_id)
          ├── dts_comments        (ticket_id, author_id)
          ├── dts_mentions        (ticket_id, mentioned_user_id, mentioned_by)
          └── dts_notifications   (ticket_id, recipient_id, actor_id)
```

| 테이블 | 주요 컬럼 |
|--------|-----------|
| `users` | id, username, password_hash, display_name, role, group_id, is_active |
| `groups` | id, name, description |
| `dts_tickets` | id, ticket_no, ticket_type, title, status, is_urgent, product_name, extra_fields(JSONB), requester_id, assignee_id |
| `dts_ticket_history` | ticket_id, from_status, to_status, reason, changed_by_id, changed_by_name |
| `dts_ticket_watchers` | ticket_id, user_id, watch_type (manual / auto_requester / auto_assignee / auto_mention) |
| `dts_notifications` | id, recipient_id, actor_id, ticket_id, type, message, is_read |
| `dts_comments` | id, ticket_id, author_id, content |
| `dts_mentions` | ticket_id, mentioned_user_id, mentioned_by |

- 사용자 테이블 FK 대상 모두 유효 (id 1~14)
- GlobalExceptionHandler에 로그 추가 후 재빌드·재배포 완료 (`04:55 UTC`)

**다음 단계**: 픽업 버튼 재클릭 → 로그 확인으로 정확한 예외 식별
